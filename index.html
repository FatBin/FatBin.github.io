simple descriptions of AndroidManifest.xml configuration

AndroidManifest.xml
用于四大组件注册的文件，可给应用程序添加权限声明

基本结构：

<?xml version=”1.0” encoding=”utf-8”?>

<application
    android:allowBackup="true"
    android:icon="@mipmap/ic_launcher"
    android:label="@string/app_name"
    android:roundIcon="@mipmap/ic_launcher_round"
    android:supportsRtl="true"
    android:theme="@style/AppTheme">
    <activity
        android:name=".First"
        android:label="The First Activity">
        <intent-filter>
            <action android:name="android.intent.action.MAIN" />
            <category android:name="android.intent.category.LAUNCHER" />
        </intent-filter>
    </activity>
</application>

分析：
在AndroidManifest.xml中，活动的注册声明要放在标签里，具体注册用标签，运行没注册过的活动会导致程序崩溃

中的和声明了该活动是项目的主活动

使用：
1、隐式Intent:
在里添加,如下





表示当前活动能够响应ACTION_START这个ACTION，附加了能够响应的Intent中带有的category，只有和都匹配Intent，该活动才会选择响应。

使用代码：
@Override
protected void onCreate(Bundle savedInstanceState) {
super.onCreate(savedInstanceState);
setContentView(R.layout.first_layout);

    Button button= (Button) findViewById(R.id.button_1);
    button.setOnClickListener(new View.OnClickListener(){

        @Override
        public void onClick(View v) {
            Intent intent=new Intent("android.intent.action.START");
            startActivity(intent);
        }
    });
}

其中因为category是DEFAULT，不用额外指明
